Explain control flow and loops using an example process from everyday life, for example 'waking up' or 'brushing your teeth'. (But not those ones):

Control flow is the order in which individual statements, instructions or function calls are executed or evaluated. The order of functions and statements matters for most programming languages, understanding it's syntax and what's best for what situation is vital. An example could be your daily routine, you wouldn't go about your day the following way: 8am: Have dinner => 9am: Get dressed for work => 10am: Go to sleep => 4pm: Wake up and go to work. => 12am: Have breakfast. While certainly you can do this, and it could be useful for a specific job that required you to work from 4pm-12am, and you weirdly like to sleep in your work clothes. What I'm trying to say is order impacts the running of your code that is simply what control flow is in a nutshell and while best practices are optimal for most scenarios understanding what your order does to your code is optimal for ensuring your website runs as intended.

Loops are pretty simple to explain. I'm gonna use the microwave to describe this one. When you're cooking food in a microwave it loops through each and every particle of food and heats them up every second. Repeating again and again and, you guessed it, again. The microwave will stop when the stop condition, the timer is reached. In programming loops it's quite similar. You have to set an initial value for the loop, a condition that the loop continues to check against, this condition will be used to evaluate when to stop doing the task. I.e. stop heating the food in the microwave, You really don't want to leave food in there for 30+ minutes. Sometimes when loops are executed without stop conditions you can get in quite the predicament and have the entirety of your code come to a halt. Pro Tip: have an ending-condition.

Explain the difference between accessing data from arrays and objects:

Ok these two are pretty different to each other. An array is a collection of a data type (or mixed data types) which are stored in a single 'list' like variable. Objects are a representation of a type of instance of a class, this can get pretty hard to describe but they're great, I promise. An example of an Object Class could be a Tree, now let's pretend to grow a tree. *poof* now you have a tree named "Steve", now Steve the tree is an instance of that class, and Steve has properties that further describe himself compared to other object and even trees. Steve leaves are green and pointy, his bark is tough and hard. That analogy might do more justice to explain it than just definitions.


Accessing Arrays:

When you access data from arrays you either will want to know what you're looking for exactly or the index location of where it would be in the array. #References are great too.

Accessing Objects:

You would want to know the property you are looking for but that might not be enough. You may want to utilize accessors and mutators in order to get data that you can't access otherwise. Objects are designed with limited data withdrawal and modifications in mind often. The reasons to limit data transfer is because you can break objects, rendering them unusable, if you modify some essential properties.
